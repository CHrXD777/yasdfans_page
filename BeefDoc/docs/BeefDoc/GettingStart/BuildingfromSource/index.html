    从源代码构建 #

    构建概述 #

    在具有可用的二进制发行版的平台上（例如 Windows），不需要从源代码构建 Beef。

    源代码可在 https://github.com/beefytech/Beef 获取。

    引导过程 #

    Beef 编译器的核心是用 C++ 编写的，而 IDE 和命令行 BeefBuild 构建系统是用 Beef 本身编写的。出于引导目的，Beef 包含一个最小的引导编译器，其唯一任务是执行初始的 BeefBuild 构建，然后执行自身的“正确”构建。

    在 Windows 上构建 #

    要求 #

    • Visual Studio 2017 或更高版本的 Microsoft C++ 构建工具。您可以从 https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022 安装 Microsoft Visual C++ Build Tools 或整个 Visual Studio 套件。
    • Platform Toolset 141 (VS2017)
    • Windows SDK 10.0.17763.0
    • CMake 3.15 或更新版本
    • Python 3.6 或更新版本
    • Git 命令行工具

    构建步骤 #

    • 执行 bin/build.bat

    请注意，这将首先下载并构建 LLVM，这需要一些时间。构建结果将位于 IDE/dist 目录中。

    在 Linux 和 macOS 上构建 #

    要求 #

    • CMake 3.15 或更新版本
    • Python 2.7
    • Git

    构建步骤 #

    • 使用 bin/build.sh 构建 Beef

    这将构建诸如 LLVM 之类的依赖项，可能需要一些时间。

    构建结果将位于 IDE/dist 目录中。

    请注意，诸如 BeefBuild 之类的命令行工具在这些平台上受支持，但 IDE 目前仅适用于 Windows。
